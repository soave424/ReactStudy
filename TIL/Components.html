Component

레고 블록 조립하듯 Components를 모아 전체 페이지를 구성함.

입력을 받아서 출력을 하는 것이 JS function과 유사하지만 입력은 Props로 받고 출력은 React elements로 나옴. 붕어빵 틀은 Component / 붕어빵은 Element

Props : (Property,속성) Component의 속성, 컴포넌트에 전달할 다양한 정보를 담고 있는 자바스트립트 객체 , 붕어빵 속 재료(팥, 슈크림 등)

Props 의 특징

1. Read-Only : 값을 변경할 수 없다. 붕어빵 만들고 난 뒤 속재료 못 바꿈. 새로운 elements로 바꾸기, 모든 리액트 컴포넌트는 Props를 직접 바꿀 수 없고, 같은 Props에 대해서는 항상 같은 결과를 보여줄 것(Pure함수)

Props의 사용법

<.Profile name="A" introduction="h" viewCount={1500}/>

꼭 기억하기

Component 이름은 항상 대문자로 시작해야 한다. 소문자는 HTML div로 인식

* Component 렌더링을 위해서는 Component로부터 Element를 만들어야 한다.

Component합성 : 여러 개의 컴포넌트를 합쳐 하나의 컴포넌트로, 복잡한 화면을 여러 컴포넌트로 나눠서 만들고 합성할 수 있음. (하위 컴포넌트를 갖음.)

Component추출 : 큰 컴포넌트를 산산조각 내기, 재사용성이 올라감. 컴포넌트가 작아질수록 기능과 목적이 명확하고 props도 단순해져서 활용도가 높아짐. 개발속도도 높아짐.

Component는 기능별로 나눠서 만드는 것이 좋음.재사용 가능한 Component를 많이 만들수록 개발 속도가 늘어남.
